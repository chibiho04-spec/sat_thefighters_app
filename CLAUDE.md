# Sat the Fighters - 案件管理アプリ

有限会社サット（映像制作会社）向け案件管理アプリ。スマートフォン向けゲーム風UI。

## プロジェクト構成

**単一HTMLファイル構成** — `案件管理アプリ.html` 1ファイルに全てのコード（HTML/CSS/JS）が完結。

```
案件管理アプリ/
├── 案件管理アプリ.html      ← 本体（編集対象）
├── CLAUDE.md                ← このファイル
└── backups/                 ← JSONバックアップ置き場（推奨）
```

## ユーザーについて

- プログラミング初心者（Claudeに自然言語で指示する形で開発）
- Mac環境で開発、ファイルはデスクトップ配下に配置
- アプリはブラウザで `file://` 直開きで動作
- 日本語でやり取り、難しい用語は避ける

## 開発ルール

- **編集は `案件管理アプリ.html` 1ファイルのみ**（HTML/CSS/JSすべてインライン）
- **編集後は必ずJSの構文チェック**してから完了する（`node --check` 等）
- 大きな変更後は localStorage キーの破壊的変更がないか確認
- 動作確認時は localStorage クリアで初期状態をテストできる
- レイアウトは **iPhone想定の phone-frame** が基本、デスクトップは @media クエリで対応
- 機能の更新追加修正などをした時は更新した場所以外もちゃんと機能しているからテストしてから完了とすること

## 重要な命名規則

- **案件番号**：`T26045` 形式（接頭辞 + 西暦下2桁 + 連番3桁）。接頭辞は **T=技術部（撮影・音声）／E=編集／P=制作（演出）** から選択（2026-06-16にD→Pへ変更）。連番は《接頭辞×年》ごとに独立し**年が変わると自動で001にリセット**（getNextOrderNum、例 2027年最初=T27001）。受注ノート(Google Sheets)と整合させるため T が正。旧 Z26xxx 仕様・自動変換は廃止済み。番号は英数字とハイフンのみ（記号は登録時に自動除去）
- **単独書類の番号**：案件に紐づかない書類は自動採番。請求書 `INV-26001`／見積書 `EST-26001`／納品書 `DLV-26001` 形式。INV- は売上表の集計対象（EST-/DLV- は対象外）。INV- からワークシートを後付け作成できる（売上表・請求書編集画面のボタン）
- **商品コード**：`Z020`、`Z104` 等（機材ごとの請求コード）
- **localStorageキー**：
  - `ws_${num}` — 各ワークシートの保存データ
  - `inv_${num}` — 請求書データ
  - `projects_saved` — 保存済み案件リスト（JSON配列）
  - `deleted_ws` — 削除済みワークシート番号
  - `equip_master` — 機材マスタ（JSON配列）
  - `last_login_name`, `last_login_time` — ログイン状態
  - `last_backup_time` — 最終バックアップ日時（ISO文字列）

## 技術的注意点

- `image-rendering: pixelated` でドット絵をシャープに表示
- `shape-rendering: crispEdges` でSVGのピクセルアートを描画
- 印刷時は `print-color-adjust: exact` でザブトン色を保持
- 案件番号は `T26xxx` 形式。旧 T26xxx→Z26xxx の自動変換（migrateStorageKeys）は廃止済み（受注ノートと整合させるため T が正）
- スタッフ・力士のアバターはSVGを `makePixelChar(palette)` で生成（共通関数）

### 💴 金額計算の正（2026-06-12確定・全箇所この式に統一すること）
- **消費税**：課税対象の明細のみ10%。`taxBase = max(0, 課税対象計 − 値引き)`、`tax = Math.floor(taxBase × 0.1)`、`合計 = (小計 − 値引き) + tax`（calcInvTotal が基準。_calcCaseSales / renderInvCards / ゴミ箱表示も同式）
- **値引き**：`status.discount` に保存（saveInvDoc / _issueInvoiceExec）。openInv 系で復元
- **外注費**：「実際に支払う税込額」で売上から控除（ユーザー決定 2026-06-12）。WS外注リスト・請求書の外注費リスト・請求書明細の外注行は各行に**課税/非課税の選択あり**（2026-06-16）。課税＝税抜入力で ×1.1、非課税＝そのまま加算。レンタルは課税扱い。`_calcCaseSales` は `osTaxableBase`(×1.1)と`osExemptSum`(そのまま)に振り分けて合算
  - 後方互換：WS外注リストの旧データ（taxable無し）は課税扱い、請求書の外注費リストの旧データ（taxable無し）と旧・単一値 `outsourceExtra` は非課税扱い（＝従来の数字を変えない）
- **受注金額（wsd-order-amount）**：**税別（本体価格）**で入力（2026-06-16）。請求書が無い案件の売上は `total = 受注額 + floor(受注額×0.1)` で税込換算して集計（請求済案件の税込と揃える）
- **明細の収集**：保存も発行も taxable / outsource / code を必ず保持する（落とすと非課税が課税に戻る事故になる）
- 印刷用HTML（buildDocHTML）に流す文字列は必ず `escapeHtml()` を通す
