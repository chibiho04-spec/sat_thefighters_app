# Sat the Fighters - 案件管理アプリ

有限会社サット（映像制作会社）向け案件管理アプリ。スマートフォン向けゲーム風UI。

## プロジェクト構成

**単一HTMLファイル構成** — `案件管理アプリ.html` 1ファイルに全てのコード（HTML/CSS/JS）が完結。

```
案件管理アプリ/
├── 案件管理アプリ.html      ← 本体（編集対象）
├── CLAUDE.md                ← このファイル
└── backups/                 ← JSONバックアップ置き場（推奨）
```

## ユーザーについて

- プログラミング初心者（Claudeに自然言語で指示する形で開発）
- Mac環境で開発、ファイルはデスクトップ配下に配置
- アプリはブラウザで `file://` 直開きで動作
- 日本語でやり取り、難しい用語は避ける

## 開発ルール

- **編集は `案件管理アプリ.html` 1ファイルのみ**（HTML/CSS/JSすべてインライン）
- **編集後は必ずJSの構文チェック**してから完了する（`node --check` 等）
- 大きな変更後は localStorage キーの破壊的変更がないか確認
- 動作確認時は localStorage クリアで初期状態をテストできる
- レイアウトは **iPhone想定の phone-frame** が基本、デスクトップは @media クエリで対応
- 機能の更新追加修正などをした時は更新した場所以外もちゃんと機能しているからテストしてから完了とすること

## ⚙️ 変更後の必須チェック工程（ユーザー要望 2026-05-27）

**コードを書き直したら、報告前に必ず以下の工程を踏むこと。**

1. **JS構文チェック**：`node --check` で確文エラーがないか確認（必須）
2. **影響範囲の確認**：
   - 変更した関数を呼んでいる他の場所（grep）をチェック
   - 関連する画面（パネル・モーダル・タブ）の表示が崩れていないか
   - z-index・position・transform の組み合わせで他要素を覆っていないか
3. **既存機能との衝突確認**：
   - イベントリスナーの重複登録なし
   - グローバル関数の上書きなし（既存名と被ったら別名にする）
   - localStorage キーの破壊的変更なし
4. **バグの自己レビュー**：
   - undefined / null になり得る箇所
   - 非同期処理（setTimeout・confirm dialog）の戻り値依存箇所
   - レスポンシブ（モバイル/デスクトップ両方）で破綻しないか
5. **報告時に明示**：「✅ 構文チェックOK」「✅ 関連箇所〇〇を確認」など、何を確認したかを書く

> ⚠️ 「動くはず」で報告しない。確認した工程を必ず明記する。

## 主な機能・タブ構成

ナビバー5タブ：
1. **🏠 ホーム** — ユーザーステータスカード、3アクションボタン、月間カレンダー
2. **📋 案件** — ワークシート一覧、編集、PDF印刷プレビュー
3. **📄 請求書** — 請求書一覧、編集、PDF印刷
4. **✊ 漢たち** — スタッフ10名（静的HTML、ドット絵アバター）
5. **🏟️ SUMO** — ミニゲーム（10力士の対戦、TV中継風UI）

その他主要画面：
- 機材マスター編集（青ヘッダー、案件編集画面から開く）
- データ管理画面（上部「データ管理」ボタンから開く・バックアップ/復元/リセット）

## 重要な命名規則

- **案件番号**：`T26039` 形式（T + 年号末2桁 + 連番3桁）※受注ノート(Google Sheets)と整合させるため T が正。旧 Z26xxx 仕様・自動変換は廃止済み
- **商品コード**：`Z020`、`Z104` 等（機材ごとの請求コード）
- **localStorageキー**：
  - `ws_${num}` — 各ワークシートの保存データ
  - `inv_${num}` — 請求書データ
  - `projects_saved` — 保存済み案件リスト（JSON配列）
  - `deleted_ws` — 削除済みワークシート番号
  - `equip_master` — 機材マスタ（JSON配列）
  - `last_login_name`, `last_login_time` — ログイン状態
  - `last_backup_time` — 最終バックアップ日時（ISO文字列）

## 既知の制約・課題

### 🔴 デスクトップ表示の問題
- 横幅が広い画面（>= 768px）でレイアウトが崩れることがある
- `applyLayout()` で JS 強制適用しているが完全には解決していない
- スタッフタブは静的HTMLで回避済み
- 新しい全画面オーバーレイを追加する時は、`@media (min-width: 768px)` の `#xxx-screen` ルールに追加必須

### localStorage の容量制限
- 全データJSONバックアップ機能で対応中（データ管理画面）
- 3日に1回の自動バックアップが動作（起動2秒後にチェック）

## 技術的注意点

- `image-rendering: pixelated` でドット絵をシャープに表示
- `shape-rendering: crispEdges` でSVGのピクセルアートを描画
- 印刷時は `print-color-adjust: exact` でザブトン色を保持
- 案件番号は `T26xxx` 形式。旧 T26xxx→Z26xxx の自動変換（migrateStorageKeys）は廃止済み（受注ノートと整合させるため T が正）
- スタッフ・力士のアバターはSVGを `makePixelChar(palette)` で生成（共通関数）
