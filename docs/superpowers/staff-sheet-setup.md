# スタッフ（漢たち）同期のセットアップ手順 — 初心者向け

作成日: 2026-05-31
対象: Sat the Fighters の「漢たち」タブのスタッフ情報を、全端末で共有（同期）する

## これは何？

「漢たち」タブのスタッフカードを**タップすると編集フォーム**が開き、名前・読み・役職・専門・稼働件数・経験年数・星・ステータス・ひとこと・役割バッジを書き換えられます。
保存すると、その内容が**専用のスプレッドシート**に送られ、**他の端末でも同じ内容が表示**されます（請求書の同期と同じしくみ）。

- アバターの**絵柄（ドット絵）は変更できません**（テキスト項目だけ編集できます）。
- スタッフは**10名固定**です（追加・削除はしません）。
- URL を設定しなければ、編集は**その端末のローカルだけ**に保存されます（同期オフでも普通に使えます）。

---

## 手順の全体像（順番が大事）

1. スタッフ用の**スプレッドシートを新規作成**する。
2. 1行目に見出し「**キー / データ / 更新日時 / 削除フラグ**」を入力。
3. 拡張機能 → **Apps Script** に `staff-sheet-Code.gs` の中身を貼り付け。
4. `SYNC_SECRET` を、**他の8シートと同じ合言葉**に書き換える。
5. **ウェブアプリとしてデプロイ**して URL を取得。
6. アプリの「データ管理 → 設定」で、**スタッフ Web App URL** に貼り付けて保存。
7. 1台で「🔄 スタッフを同期」を押す → シートに10名分がシードされる。
8. 他の端末はアプリを開くと（起動約1.5秒後に）自動で取り込み。

---

## 1. スプレッドシートを作る

1. Google ドライブで新しいスプレッドシートを作成（名前は「スタッフ同期」など）。
2. 1行目に、左から順に次のとおり入力します。

| A1 | B1 | C1 | D1 |
|----|----|----|----|
| キー | データ | 更新日時 | 削除フラグ |

> 見出しが空でも、初回アクセス時にスクリプトが自動で入れます。手で入れておくと安心です。

## 2. Code.gs を貼り付ける

1. メニュー「拡張機能 → Apps Script」を開く。
2. 既存の `myFunction` などを全部消して、`staff-sheet-Code.gs` の中身を**丸ごと貼り付け**。
3. 上の方にある `var SYNC_SECRET = 'ここに合言葉';` を、**他のシートと同じ合言葉**に書き換える。
   - 例：`var SYNC_SECRET = 'sat-fighters-2026-aBcD';`
   - 合言葉は GitHub には載せない。アプリの設定画面で各端末に1回入力するもの。

## 3. デプロイ（ウェブアプリとして公開）

1. 右上「デプロイ」→「**新しいデプロイ**」。
2. 種類の歯車 → 「**ウェブアプリ**」。
3. 設定：
   - 説明：任意（例「スタッフ同期 v1」）
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
4. 「デプロイ」→ 認証を求められたら自分の Google アカウントで許可。
5. 表示された **ウェブアプリ URL**（`https://script.google.com/macros/s/.../exec`）をコピー。

## 4. アプリに URL を登録

1. アプリ上部の「データ管理」→ 設定画面を開く。
2. 「**スタッフ Web App URL**」欄に、コピーした URL を貼り付け。
3. 「💾 URLを保存」を押す。
4. 同じ画面の「🔑 同期の合言葉」に、手順2で決めた合言葉が入っているか確認（未入力なら入れる）。

## 5. 最初のシード（1台だけで実行）

1. 設定画面の「**🔄 スタッフを同期**」を押す。
2. 「✅ 送信 10件・取得 ◯件」と出れば、シートに10名分が書き込まれました。
3. 以降は、各端末でカードをタップ → 編集 → 保存すると自動でシートへ送信され、他端末は起動時に取り込みます。

---

## 動作確認

1. 1台でスタッフカードをタップ → 役職などを変えて「💾 保存する」。
   - スプレッドシートの「データ」列の JSON が更新されればOK。
2. 別の端末でアプリを開く → 起動して少しすると、その変更が反映される。
3. わざと**間違った合言葉**を入れて同期 → 失敗する（拒否される）ことを確認 → 正しい合言葉に戻す。

## しくみ・既知の限界

- 同期方式は請求書と同じ「**更新日時で後勝ち（Last-Writer-Wins）**」。
  - 同じスタッフを2台でほぼ同時に編集すると、**後から保存した方**の内容が残ります。
- スタッフは10名固定。シートに知らない「キー」があってもアプリは無視します。
- ネット不調で送信に失敗しても、**未送信として記録**され、起動時や「🔁 いま再送する」でまとめて再送されます。
- 合言葉は URL クエリに乗るため、GAS の実行ログ等に残りえます（内部ツールとしては許容）。
