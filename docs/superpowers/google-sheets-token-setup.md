# 同期の合言葉（token）設定手順 — 初心者向け

作成日: 2026-05-31
対象: Sat the Fighters の Google スプレッドシート同期（親/子/取引先/商品/メディア/機材/請求書/運転記録の各 Web App）

## これは何？

GitHub Pages は**公開**サイトなので、`index.html` に書いた内容は誰でも見られます。
URL を隠すだけでは守れないため、**サーバ側（Google Apps Script）でリクエストに「合言葉」が付いているか検査して、合っていなければ拒否**します。

- 合言葉は**アプリのコードには書きません**。各社員が設定画面で1回だけ入力します（その端末の localStorage に保存）。
- アプリは `script.google.com` 宛の通信に、自動で `?token=合言葉` を付けて送ります（実装済み）。
- 各 Web App の `Code.gs` に、下の「ガード」を追加して合言葉を検査します。

> ⚠️ これは「内部ツール相応」の対策です。合言葉を知っている人は読み書きできます。合言葉を外部に貼らない運用が前提です。

---

## 手順の全体像（順番が大事）

1. **アプリは更新済み**（合言葉の入力欄と自動付与は実装・反映済み）。
2. 合言葉を1つ決める（例：`sat-fighters-2026-aBcD`。推測されにくい文字列にする）。
3. **社員へ合言葉を配る**（LINE や口頭など。GitHub には載せない）。
4. 各社員が、アプリの「データ管理 → 🔑 同期の合言葉」に入力して保存。
5. **全員の入力が終わってから**、各 `Code.gs` にガードを追加して再デプロイ（=サーバ側で検査ON）。
   - 先にサーバ側をONにすると、まだ合言葉を入れていない端末が同期できなくなるため、**順番を守る**。

---

## Code.gs に追加するガード（8個すべてに同じものを入れる）

各 Web App の `Code.gs` の**いちばん上**に、次を貼り付けます。

```javascript
// ===== 合言葉チェック =====
var SYNC_SECRET = 'ここに合言葉';  // ← アプリに入れる合言葉と「完全一致」させる（前後の空白に注意）

function _syncAuthOK(e) {
  var t = (e && e.parameter && e.parameter.token) ? String(e.parameter.token) : '';
  return t === SYNC_SECRET;
}
function _syncDeny() {
  return ContentService
    .createTextOutput(JSON.stringify({ ok: false, error: 'unauthorized' }))
    .setMimeType(ContentService.MimeType.JSON);
}
```

次に、既存の `doGet(e)` と `doPost(e)` の**最初の行**に検査を1行ずつ足します。

```javascript
function doGet(e) {
  if (!_syncAuthOK(e)) return _syncDeny();   // ← この1行を先頭に追加
  // …（ここから下は今までの処理。さわらない）…
}

function doPost(e) {
  if (!_syncAuthOK(e)) return _syncDeny();   // ← この1行を先頭に追加
  // …（ここから下は今までの処理。さわらない）…
}
```

ポイント：
- `SYNC_SECRET` の値は、アプリの設定画面に入れる合言葉と**1文字も違わない**ようにする。
- GET でも POST でも、合言葉は URL の `?token=...` で届きます（GAS は `e.parameter.token` で両方読めます）。アプリ側がそうやって送ります。

---

## 再デプロイ（変更を有効にする）

`Code.gs` を編集しただけでは反映されません。**URL を変えずに**更新するため、次の手順で行います。

1. Apps Script エディタ右上「デプロイ」→「**デプロイを管理**」。
2. 今動いている Web App デプロイの**鉛筆（編集）**を押す。
3. 「バージョン」を「**新バージョン**」にして「デプロイ」。
   - こうすると **URL は同じまま**、中身だけ新しくなります（新規デプロイにすると URL が変わってしまうので注意）。
4. これを 8 個の Web App すべてで行う。

---

## 動作確認

1. 合言葉を入れた端末でアプリを開く → 案件・請求書などが今までどおり同期できればOK。
2. わざと**間違った合言葉**を入れて保存 → 取り込み・送信が失敗する（拒否される）ことを確認 → 正しい合言葉に戻す。
3. ブラウザで `Web App の URL/exec?action=list`（token なし）を直接開く → `unauthorized` が返れば、部外者は弾けています。

---

## 合言葉を変えたいとき（流出時など）

1. 新しい合言葉を決める。
2. 8 個の `Code.gs` の `SYNC_SECRET` を新しい値に書き換え → それぞれ再デプロイ。
3. 全社員に新しい合言葉を配り、各端末の設定画面で入れ直す。

---

## 補足・既知の限界

- 合言葉は URL クエリに乗るため、GAS の実行ログやブラウザ履歴に残りえます（内部ツールとしては許容）。
- 本当に強固にするなら、Web App を「会社の Google ドメインのみアクセス可」にして OAuth ログイン必須にする方法があります（アプリの作り替えが必要なため今回は採用せず）。
- アプリ側 localStorage キー：`gas_token`（この端末のみ・コードには未保存）。
