# 紹介モード（PINログイン＝ダミーデータのデモ）設計仕様

作成日：2026-07-04 / 対象：`案件管理アプリ/index.html`（単一HTMLファイル）

## 1. 目的
社外の人にアプリを紹介するとき、実データ（取引先・金額・案件）を一切見せずに、**アプリの全機能をダミーデータで実演できる「紹介モード」**を作る。入口は既存のPINログイン。

## 2. 決定事項（ユーザー確認済み 2026-07-04）
- **PINログイン＝常に紹介モード**。Googleログイン＝実データ（従来どおり）。
- **スタッフ名は実名のまま**（漢たちタブ・クルー名・ログインの名前選択は変更しない）。
- **案件・取引先・金額などのデータは全部ダミー**。
- **経営数値もダミーに切替**：紹介モード中は損益分岐500万円・目標粗利600万円（実モードは750万/800万のまま）。
- 紹介モード中の**編集は自由**（メモリ上のみ。終了・タブを閉じる・再読み込みで消える）。

## 3. アーキテクチャ（案A：起動時にlocalStorageを丸ごと差し替え）
本体コード（データ読み書きの数百箇所）は**一切変更しない**。起動の最初でデータ保存庫そのものを差し替える。

### 3-1. フラグとライフサイクル
- フラグ：`sessionStorage.setItem('demo_mode','1')`（**sessionStorage** なのでタブを閉じれば自動消滅。他タブ・実データに影響なし）。
- 入口：`doLocalLogin()` でPIN照合成功 → `demo_mode=1` をセットして `location.reload()`。
  - file:// のPIN省略ログインも同様に紹介モードになる（＝**file://やPINから実データに到達する経路は無くなる**。実データはGoogleログインのみ。これは意図した仕様変更でセキュリティ向上）。
- 起動：メイン`<script>`（index.html:6972）の**先頭**で `demo_mode` を判定。ONなら本体コードが動く前に偽ストレージをインストール。
- 出口：`doLogout()`（⏏退出）で `demo_mode` を除去して `location.reload()` → 通常のログイン画面へ。再読み込み（F5）でもダミーデータは初期状態に戻る（メモリなので）。

### 3-2. 偽ストレージ（DemoStorage）
- プレーンオブジェクトに `getItem/setItem/removeItem/key/clear` を**非enumerableメソッド**として定義し、データは通常プロパティで保持 → `Object.keys(localStorage)`（13箇所）・`localStorage.key(i)`・`.length` が本体コードのまま自然に動く。
- `Object.defineProperty(window, 'localStorage', { value: fake, configurable: true })` で差し替え（Chrome/Safari対応。インストール直後に読み書き＋enumerationのセルフテストを行い、失敗したらエラーバナーを出して紹介モードを中止＝実データ画面には進まない）。
- **同期キーの特別扱い（最重要）**：`gas_url` / `gas_url_seed` / `gas_url_driving` / `gas_token` は `getItem`で常に `null` を返し、`setItem` を無視する（起動時の `GAS_URL_DEFAULT` 強制シードも無効化される）。
- インストール時にダミーデータ（§4）をシードする。`last_login_name`（PINで選んだ名前）と `last_login_time=今` もシード → checkAutoLogin がそのまま通ってホームへ直行。

### 3-3. 同期・外部送信の二重ブロック
1. 偽ストレージが同期URLを常に空にする（上記）→ すべての同期関数は「URL未設定」として何もしない。
2. さらに `window.DEMO_MODE===true` のとき `pushKind` / `pullKind` / `syncAll` / 勤務時間(worklog)取得 の先頭で即return（ベルト＆サスペンダー）。
3. 自動バックアップ：`last_backup_time` を未シード（null）→ `checkAutoBackup` は何もしない仕様のためダウンロードは発生しない。手動バックアップはダミーJSONが落ちるだけなので許容。
4. 天気予報（open-meteo）など**個人情報を含まない外部読み取りAPI**は許容（動くデモとして見せる価値があるため）。

### 3-4. 経営数値の切替
`BREAK_EVEN` / `TARGET_SALES`（index.html:13749-13750）を `window.DEMO_MODE ? 5000000 : 7500000` / `? 6000000 : 8000000` に変更。

### 3-5. 見た目（誤解防止）
- 上部apiバーのラベルを「🎭 紹介モード（ダミーデータ・同期オフ）」の常時表示に差し替え（クリックでデータ管理を開く挙動は維持）。同期失敗の赤警告は紹介モード中は出さない。
- ログイン画面の「PINでログイン（お試し用）」→「**🎭 紹介モード（デモを見る）**」に文言変更。PIN入力UI・名前選択はそのまま。

## 4. ダミーデータ（シード内容）
**撮影日は「今日」を基準に相対配置**して、いつ開いてもカレンダー・ダッシュボード・撮影予定が生きて見えるようにする。

| kind | 内容 |
|---|---|
| projects_saved + ws_ | 約12件。架空の取引先×件名（例：やんばるテレビ「観光PR番組 撮影技術」、(株)シーサー広告「新商品CM撮影」、美ら海メディア「企業VP」、首里映像社「MV撮影」、ちゅら企画「イベント収録」等）。撮影日：過去2週〜未来3週に分散（未来5件以上→ダッシュボード映え）。クルーは実名スタッフを割当。機材チェック・メモ入りも数件 |
| inv_ | 請求書 約10件。過去5ヶ月に請求済/入金済を分散（売上グラフ・粗利が毎月立つ。月200〜600万円程度のダミー金額）＋未請求2件＋見積書1件。明細は複数行・非課税/外注行・行備考・空行入り（新機能も見せられる） |
| equip_master | ダミー機材 約15点（カメラ/レンズ/照明/音声/メディア等、実在モデル名は可・価格はダミー） |
| client_master_cache | 上記の架空取引先 5〜6社（住所・担当者もダミー） |
| product_master_cache / media_cache | 各数件のダミー |
| inv_templates | 「定期番組 技術協力（月次）」等 1〜2件 |
| 簡易ワークシート | 1〜2件 |
| last_login_name / last_login_time | PINで選んだスタッフ名／現在時刻 |

- 受注番号は実データと同じ体系（T26xxx/E26xxx/P26xxx）で本物らしく。
- シードは `_demoSeed()` 関数として実装し、日付は `new Date()` からの相対計算で生成。

## 5. 影響範囲（壊さない）
- 本体のデータ読み書き・画面コードは無変更。変更点は①script先頭のシム＋シード（追加）②doLocalLogin（数行）③doLogout（数行）④BREAK_EVEN/TARGET_SALES（1行×2）⑤同期関数のDEMOガード（各1行）⑥apiバー表示＋ログイン文言。
- Googleログイン・自動ログイン（実セッション）・#dashboard モニターモードは従来どおり実データで動く。
- 実モードの動作は `demo_mode` フラグが無い限り1バイトも変わらない（シムはインストールされない）。

## 6. テスト計画（CLAUDE.md必須工程）
1. **実データ非汚染**：紹介モード起動→操作（新規案件作成・請求書発行・削除）→終了、の前後で実localStorageの全キー・全値が完全一致（スナップショット比較）。
2. **通信ゼロ**：紹介モード中、script.google.com への fetch が発生しない（pushKind/pullKind監視＋ネットワークログ）。
3. **見栄え**：ホーム（カレンダーに案件が載る）・ダッシュボード・売上表（グラフが立つ・損益分岐500万/目標600万表示）・請求書一覧/編集/プレビュー・機材・受注ノートが全てダミーで表示される。
4. **編集の揮発**：デモ中の編集→再読み込みで初期ダミーに戻る。退出→通常ログイン画面→Googleセッションの実データは無傷。
5. **実モード非回帰**：demo_modeなしで従来どおり（同期・保存・自動ログインが正常）。
6. **UI4観点**：スマホ390px/PC × 文字100%/150%（紹介モードバッジ・ログイン画面の崩れなし）。

## 7. やらないこと（YAGNI）
- スタッフ名のダミー化（実名のままでOKと確認済み）。
- デモ専用の操作ガイド・ツアー機能。
- デモデータのカスタマイズUI（シードはコード内固定）。
- 紹介モード専用URL（?demo=1等）——入口はPINログインに一本化。必要になったら後で追加可。
