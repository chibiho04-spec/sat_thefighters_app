# キャラクター詳細ステータス画面 設計

最終更新：2026-06-02

## 目的

ホームのキャラクターカード（および「漢たち」タブの10名一覧）をタップすると、その人の
リッチな詳細ステータス画面を開けるようにする。自分（ログインユーザー）は編集も可能、
他人は閲覧のみ。これにより「漢たち」タブを廃止せず、中身を強化する。

## 全体方針（ブレストで確定した内容）

- **タブ構成：案B**（「漢たち」タブは残す）。タブの10名一覧はそのまま。タップ先を
  「編集フォームだけ」から「詳細ステータス画面（＋自分なら編集）」に強化する。
- ホームの自分カードをタップ → 自分の詳細画面。
- 「漢たち」タブの各カードをタップ → その人の詳細画面。
- ナビバーは現状の5タブのまま（🏠ホーム / 📋案件 / 📄請求書 / ✊漢たち / 🏟️SUMO）。

### ヘッダ左カラム（縦並び）

- ドット絵アバター（大）
- 名前（姓＋名）
- 英語表記
- 役職（役割：「C / CA カメラマン」など）
- 区切り線の下に：入社年月日・勤続年数

### ヘッダ右カラム（ゲームステータス）

- **Lv ＋ ★評価**（同じ行に並べる）
- EXPバー（数値 現在/次レベル）
- 筋力 / 素早さ / 粘り強さ のバー＋数値

### 中段：勤務データ（※自分＝ログインユーザー本人のときだけ表示）

- 月間勤務時間（作業日報シート）
- 残業時間（作業日報シート）
- 進行中案件数（当月クルー参加件数）

他人の画面では勤務データブロックは丸ごと非表示。

### 下段：担当案件リスト

- その人がクルーに入っている保存済み案件の一覧（案件名・撮影日付）。
- タップでその案件を開く（案件編集画面へ遷移）。

### 表示しないもの（ブレストで除外）

- ひらがな読み（「しろま しん」）→ 載せない
- スキル（📷速写など）→ 載せない
- 部門別の実績（5）→ 載せない
- ひとこと/メモ（7）→ 載せない

## データソース（既存コードを再利用）

| 表示項目 | 取得元 |
|---|---|
| 名前・英語・役職・入社日・勤続・★ | `loadStaffMaster()`（localStorage `staff_master`）／`getStaffByKey(key)` |
| アバター（ドット絵） | `makePixelChar(SUMO_FIGHTERS[i].palette)`（STAFF_KEYS の index で対応） |
| Lv・筋力・素早さ・粘り | `SUMO_FIGHTERS[i]`（level/str/spd/end） |
| EXP（現在/次） | `USER_STAFF[key]`（exp/expNext）。**shin にしか無い**ため、無い人は EXPバーを非表示にし Lv のみ表示 |
| 月間勤務・残業（※自分のみ） | 既存 worklog 機能（`USER_STAFF[key].hours/overtime`、作業日報シート由来） |
| 進行中案件数（※自分のみ） | 保存済み案件のうち当月かつ本人がクルーの件数（`getSavedProjects()` ＋ `isUserInEvent` 系の姓マッチ） |
| 担当案件リスト | `getSavedProjects()` を本人クルーで絞り込み（`isUserInEvent(ev)` のロジックを流用） |

- スタッフキー ↔ index は既存 `STAFF_KEYS` 配列で対応（SUMO_FIGHTERS と同順）。
- 「本人かどうか」は `localStorage('last_login_name')` と開いた人の key を比較して判定。

## 画面の作り（実装方針）

- 既存の `#staff-edit-screen`（右からスライドインするフルスクリーンオーバーレイ）と
  同じパターンで、新規オーバーレイ `#staff-detail-screen` を1つ追加する。
- `openStaffDetail(key)` で開く。中で上記データを集約して描画。
  - 自分のときはヘッダに「編集」ボタンを出し、既存の `openStaffEdit(key)` を呼ぶ（編集UIは流用・新規作成しない）。
  - 他人のときは編集ボタンを出さない（閲覧のみ）。
- タップ動線の変更：
  - 「漢たち」タブのカード `onclick`：今は `openStaffEdit(s.key)` → これを `openStaffDetail(s.key)` に変更。
  - ホームの自分カード `#user-staff-card`：クリックで `openStaffDetail(<自分のkey>)` を開くようにする（今はクリック不可）。
- 閉じるボタンと、既存のオーバーレイ閉じ管理（`#staff-edit-screen` を登録している配列・
  ESC/戻る処理）に `#staff-detail-screen` を同様に登録する。
- デスクトップ対応：`@media (min-width:768px)` の `#xxx-screen` ルール群に
  `#staff-detail-screen` を追加（既知の制約：新規オーバーレイは必ず追加が必要）。

## エラー処理・エッジケース

- staff_master が未取得でも `_defaultStaffMaster` にフォールバック（既存 `loadStaffMaster` がやる）。
- SUMO_FIGHTERS の index が見つからない人は str/spd/end を「-」表示にして落ちないようにする。
- EXP が無い人（shin 以外）は EXPバーを非表示、Lv のみ表示。
- 勤務データ・進行中件数は「自分の画面」以外では DOM ごと非表示（他人のデータは持っていない）。
- 担当案件が0件なら「担当案件はありません」を表示。
- 星は表示のみ（現状 編集不可の仕様を踏襲）。

## テスト観点（実装後に確認）

1. JS構文チェック（`node --check`）。
2. ホームの自分カードタップ → 自分の詳細が開く。編集ボタンあり → `openStaffEdit` が開く。
3. 漢たちタブの他人カードタップ → その人の詳細が開く。勤務データ非表示・編集ボタン無し。
4. 担当案件タップ → 該当案件が開く。
5. 詳細→閉じる→他タブが正常表示（オーバーレイが残らない）。
6. デスクトップ幅でレイアウトが崩れない。
7. EXPの無い人でエラーが出ない。
