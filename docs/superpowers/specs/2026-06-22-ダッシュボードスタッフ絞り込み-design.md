# ダッシュボード スタッフ絞り込み（誰がどの案件か色分け）設計仕様

作成日：2026-06-22 / 対象：`案件管理アプリ/index.html`

## 1. 目的
経営陣の「その日、誰がどの案件に出ているか」を一目で把握する。ダッシュボードにスタッフ選択（複数可）を付け、選んだスタッフが担当する案件を**スタッフごとの色**で示す。

## 2. 決定事項（ユーザー確認済み 2026-06-22）
- **選択UI**：スタッフ10名のボタンを上部に**横並び・常時表示**、タップでON/OFF（複数選択）。
- **色の付け方**：**スタッフごとに色分け**。1案件に複数の選択スタッフがいれば複数色を並べる。

## 3. データ
- スタッフ名（10）：`SUMO_FIGHTERS` の name（川崎・上原・平岡・川野・城間・新崎・玻座真・森田・上地・松田）。順序は `STAFF_KEYS` と一致。
- 案件のクルー：`getWsDisplayData(num)` の crew*（crewProducer/crewD/crewPm1/crewPm2/crewC/crewA/crewA2/crewSound/crewEdit）。値はスタッフ名。
- 判定 `_caseStaffNames(num)`：上記crew値のうち、スタッフ10名に一致する名前の集合。
- 色 `_dashStaffColor(name)`：10色の固定パレットを STAFF index で割当。

## 4. UI / 挙動
- 状態 `_dashStaffSel`（選択スタッフ名の Set）。`toggleDashStaff(name)` でトグル→`renderDashboard()` 再描画。「クリア」ボタンで全解除。
- **スタッフバー**（`.dash-staffbar`、dash-body 先頭・flex:0）：各ボタンは選択中は割当色で塗り、非選択は白系。
- **未選択時**：従来表示（色付けなし）。
- **1名以上選択時**：
  - **ワークシートカード**：そのカードに居る「選択スタッフ」を色チップ（● 名前）でヘッダーに表示。選択スタッフが1人も居ないカードは薄く（opacity 0.3）。
  - **カレンダーのチップ**：その案件に選択スタッフが居れば、チップを担当スタッフ色で塗る（複数なら色を縦ストライプ）。居なければ薄く。

## 5. 実装（追加中心・既存は流用）
- 追加：`_dashStaffPalette`、`_dashStaffColor`、`_caseStaffNames`、`_dashStaffSel`、`toggleDashStaff`、`renderDashStaffBar`。
- `buildWsPreviewHTML(num, conflictIds)` に「選択スタッフ色チップ」表示と dim 反映を追加（選択状態は global を参照）。
- `renderDashCalendar` のチップ生成に色/ dim を反映。
- `renderDashboard` で `renderDashStaffBar()` を呼ぶ。
- 読み取り専用（表示のみ）。既存の重複警告・フィット・クリック編集は不変。

## 6. テスト
- スタッフ未選択＝従来表示。1名選択＝その人の案件だけ色＋他は薄く、カレンダーチップも色。複数選択＝各色。クリアで解除。
- 1案件に複数選択スタッフ→複数色。クリックで編集遷移が従来どおり。構文OK・実機4観点。

## 7. やらないこと
- スタッフの稼働集計・カレンダー以外への展開・欠席/休暇管理。色分け表示のみ。
