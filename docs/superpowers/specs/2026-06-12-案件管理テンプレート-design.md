# 案件管理テンプレート（まっさら版）設計書

作成日：2026-06-12
ステータス：ユーザー承認済み（壁打ちで確定）

## 目的

サット版の案件管理アプリ（index.html）を元に、**他の会社でも使える「まっさらなテンプレート」**を新規作成する。
将来、別の会社に渡すときは、このテンプレートをコピーして shin さんが Claude に指示してカスタマイズ版を作る。

## 壁打ちで決まったこと

| 論点 | 決定 |
|------|------|
| 誰がカスタマイズするか | shin さんが Claude で会社ごとに作り分ける（設定ウィザードは作らない） |
| ゲーム風の要素（SUMO・ドット絵・レベル表示） | 見た目・機能は残し、中身（名前・データ）だけ差し替え可能にする |
| Google スプレッドシート同期 | テンプレートに組み込んでおく。接続先 GAS URL だけ会社ごとに設定。**URL が空なら同期オフで動く** |
| 想定業種 | 特になし（将来備え）。「機材」などの画面の言葉はそのまま。言葉の変更は会社ごとに Claude で個別対応 |
| 作り方 | 案1「コピーして抜く」方式。今のアプリをコピーし、サット固有要素を除去して設定ゾーンに集約 |
| サンプルデータ | 説明用にワークシート・請求書などの架空サンプルを数件入れておく |

## 1. 置き場所

今のサット版アプリには一切手を付けない。**新しいフォルダ＋新しい Git リポジトリ**を作る。

```
Desktop/claudecode/sat/
├── 案件管理アプリ/        ← 今のサット版（変更しない）
└── 案件管理テンプレート/   ← 今回作る まっさら版
    ├── index.html         ← テンプレート本体（サット版のコピーから改造）
    ├── CLAUDE.md          ← テンプレート用の開発ルール
    ├── カスタマイズ手順.md  ← 「新しい会社版の作り方」手順書
    ├── docs/              ← GAS 同期コード（unified-sheet-Code.gs）と設置手順
    └── backups/           ← JSON バックアップ置き場
```

## 2. 会社設定ゾーン（COMPANY_CONFIG）

index.html の先頭近くの script に、会社ごとに書き換える項目を **1ブロックに集約**する。
「ここだけ書き換えれば会社版が完成する」が理想形。

含める項目：

- アプリ名・会社名
- 案件番号の頭文字（サットは `T` → 会社ごとに変更可。例 `A26001`）
- 単独書類の頭文字（`INV-` / `EST-` / `DLV-` はそのまま既定値）
- 請求書に載る発行者情報（社名・住所・電話・振込先・インボイス登録番号）
- ログインユーザーと 4桁 PIN（USERS / USER_STAFF 相当）
- スタッフ一覧（名前・役職・アバターの palette 設定）
- 同期先の GAS Web App URL（空文字なら同期機能オフ）

## 3. 空にするデータ・仮データ

- 機材マスタ・取引先・商品・案件・請求書の**サット実データは全部除去**
- スタッフは**仮スタッフ3名**（例：サンプル太郎など）だけ入れて、差し替え例として使う
- SUMO の力士はスタッフ一覧から自動生成する（別途10名定義しない）
- サットの振込先・住所・shin のログイン情報は除去し、サンプル値（admin / 0000 など）に置き換え

## 4. 漢たちタブのデータ駆動化（一番の改造ポイント）

現状の漢たちタブはスタッフ10名分の静的 HTML 直書き。テンプレートでは
**COMPANY_CONFIG のスタッフ一覧から自動生成**する方式に変える。
スタッフの追加・削除が設定の書き換えだけで済むようになる。
アバターは既存の `makePixelChar(palette)` を使う（共通関数のまま）。

## 5. 説明用サンプルデータ

初回起動時（localStorage が空のとき）に、架空のサンプルデータを投入する。

- **ワークシート3件**：
  1. 請求書発行まで終わった完了案件
  2. 機材・スタッフが入った進行中案件
  3. シンプルな小さい案件
- **請求書2〜3件**：サンプル案件に紐づくもの＋単独請求書（INV- 形式）1件。売上表にも数字が出る状態にする
- **機材マスタ3〜5件・取引先2社程度**：サンプル案件が参照する分だけ
- 名前はすべて架空とわかるもの（「株式会社サンプル商事」「テスト株式会社」など）
- **本番利用開始時**は既存の「データ管理 → リセット」でサンプルを全消去できる。手順書に明記する

## 6. カスタマイズ手順.md（手順書）

新しい会社版を作るときの流れをチェックリスト形式で書く：

1. テンプレートフォルダをコピーして会社名のフォルダを作る
2. COMPANY_CONFIG を書き換える（Claude への指示例文も載せる）
3. 会社専用のスプレッドシートを作り、GAS（unified-sheet-Code.gs）をデプロイ
4. GAS URL を設定ゾーンに入れる（同期しない運用なら空のまま）
5. データ管理 → リセットでサンプルデータを消す
6. 動作確認（下記の検証項目）

## 7. 検証・完了条件

- `node --check` で JS 構文エラーなし
- 「サット」「Sat the Fighters」「城間」「shin」「T26」「川崎」など**サット固有ワードを grep で総ざらい**して痕跡ゼロを確認（SUMO 等の残す機能名は除く）
- localStorage クリア → 初回起動でサンプルデータが入り、各画面（ホーム・案件・請求書・漢たち・SUMO・売上表・データ管理）が正常表示
- UI 検証マトリクス：スマホ幅(390px)／デスクトップ幅 × 文字サイズ100%／150% の4パターン
- GAS URL 未設定の状態で同期エラーが出ない（静かにオフ）
- 金額計算はサット版の「金額計算の正」（calcInvTotal 基準）をそのまま維持

## やらないこと（スコープ外）

- アプリ内の初期設定ウィザード（カスタマイズは Claude で行うため不要）
- 「機材」などの画面ラベルの設定化（会社ごとに Claude で個別対応）
- 本番サット版アプリの改造（テンプレートは完全に別フォルダ・別リポジトリ）
- ゼロからの作り直し
- スマホアプリ化（ブラウザ版チームの担当範囲。チーム編成メモ参照）
