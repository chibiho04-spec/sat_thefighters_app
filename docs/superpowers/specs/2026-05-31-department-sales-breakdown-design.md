# 3部門別売上 専用画面 — 設計ドキュメント

作成日: 2026-05-31
対象アプリ: Sat the Fighters（案件管理アプリ / 単一HTML構成 `index.html`）

## 背景・目的

社長から「**編集・撮影・プロデュース**の3部門で、売上がどう配分されているか知りたい」という要望。
各案件の売上を3部門に仕分けし、部門ごとの売上規模と構成比を見える化する専用画面を作る。

## スコープ

- 対象：**3部門別売上の集計・表示**のみ。
- 非対象（別機能・後回し）：機材のリース/ローンの月次ペイバック管理（別途設計）。
- 同期対象外：請求書・スタッフは従来どおり。

## 部門の定義（役割→部門のマッピング）

ワークシートのクルー役割を3部門へ対応させる（ユーザー確認済み）。

| 部門 | 含む役割 |
|------|----------|
| プロデュース | P（プロデューサー）/ D（監督）/ PM |
| 撮影 | C（カメラマン）/ CA / S（音声）/ 機材費 |
| 編集 | E（編集） |

- 「機材費（撮影機材一式）」は**撮影部門**に計上する（ユーザー要望）。

## 集計の基準額

- 部門別に出すのは「**売上**」＝ `請求額 − 外注費`（既存 `_calcCaseSales` の `netInhouse` を踏襲）。

## 計算ロジック

### A. 明細がある案件（実額ベース）

各案件の請求明細（`status.lines`、無ければ `master.lines`）を1行ずつ分類する。
判定は行の `name`（役割ラベル）に含まれるキーワードで行う。

| キーワード（name に含まれる） | 部門 |
|-------------------------------|------|
| プロデュース | プロデュース |
| 企画・演出 / 監督 | プロデュース |
| プロダクションマネージャー | プロデュース |
| カメラマン | 撮影 |
| 撮影助手 | 撮影 |
| 音声 | 撮影 |
| 撮影機材 | 撮影 |
| 編集 | 編集 |
| 上記いずれにも該当しない自由入力行 | 未分類 |

注意点:
- `outsource`（外注）チェックが入っている行は**集計から除外**（コスト扱いのため）。
- 各行のグロス額＝ `qty × price`（税抜）。
- 部門グロス＝その部門に分類された非外注行の合計。

#### 売上（net）への落とし込み — 比率按分

部門の取り分（売上）は、グロスの構成比 × 案件の売上（外注費控除後）で求める。

```
部門売上 = netInhouse × (部門グロス ÷ 全非外注行グロス合計)
```

- これにより各部門の合計が案件の売上（`netInhouse`）にピッタリ一致する。
- 外注費による減額は、各部門に比率どおり配分される。

エッジケース:
- 全非外注行グロス合計が 0 の場合（按分不能）は、その案件は「未分類」に売上全額を入れる。

### B. 明細が無い案件（役割人数で推定按分）

請求明細が未作成の案件（`_collectSalesItems` のフォールバックで `wsOrderAmount` を使うもの）は、
クルー欄の役割の**人数比**で売上（net）を按分する（推定）。

```
部門人数 = その部門に属する役割欄の入力人数
  プロデュース: crew_producer(1) + crew_d(1) + crew_pm1(1) + crew_pm2(1)
  撮影:        crew_c(1) + crew_a(1) + crew_a2(1) + crew_sound(1)
  編集:        crew_edit(1)
部門売上 = netInhouse × (部門人数 ÷ 全部門人数合計)
```

エッジケース:
- 全部門人数合計が 0（クルー未入力）の場合は「未分類」に売上全額を入れる。

### 期間・フィルタ

- 案件の月割りは既存 `_parseDateForSales(it.date)` を流用（売上表と同じ基準日）。
- 期間切替：`全期間` / `年（西暦）`。売上表の期間UIと整合させる。

## 画面設計（専用フルスクリーン画面）

### 入口
- **売上表（sales-screen）から独立画面として開く**ボタンを追加。

### 構成
1. ヘッダー（戻るボタン、タイトル「部門別売上」、期間切替）
2. サマリー
   - 3部門の売上合計と構成比を**横帯バー**で表示（プロデュース／撮影／編集／未分類）
   - 各部門の金額と％
3. 部門ごとの内訳
   - 部門を選ぶ or アコーディオンで、貢献した案件のリスト（案件名・受注No・その部門への配分額）
4. （余裕があれば）月別推移
   - 月ごとの3部門売上の積み上げ表示

### 配色（部門カラー案）
- プロデュース：紫系
- 撮影：青系
- 編集：緑系
- 未分類：グレー

## 実装上の制約・注意（CLAUDE.md 準拠）

- 編集対象は `index.html` 1ファイルのみ（HTML/CSS/JS インライン）。
- 新しい全画面オーバーレイを追加するので、`@media (min-width: 768px)` の `#xxx-screen` ルールに**必ず追加**する（デスクトップ表示崩れ防止）。
- 既存関数の再利用：`_collectSalesItems()` / `_calcCaseSales()` / `loadInvStatus()` / `loadWS()` / `loadEquipMaster()` / `_parseDateForSales()`。
- 既存のグローバル関数名と衝突しない新規名を使う（例：`renderDeptSalesScreen` / `openDeptSales` / `_collectDeptBreakdown`）。
- localStorage キーの破壊的変更なし（読み取りのみ）。
- 編集後は JS 構文チェック（インラインJS抽出 → `node --check`）。
- ブラウザ実測で関連画面（売上表・既存オーバーレイ）が崩れないか確認。

## データ依存・前提

- 部門の正確な実額は「請求明細を作っている案件」でのみ得られる。明細なしは役割人数での推定。
- クルー欄・機材選択が入力されている前提（既に入力UIあり）。

## 成功基準

- 売上表から専用画面を開ける。
- 3部門（＋未分類）の売上合計・構成比が表示される。
- 部門合計が、対象期間の全案件の売上合計と一致する（按分により保証）。
- デスクトップ／モバイル両方でレイアウトが崩れない。
- 既存機能（売上表・請求書・ワークシート）に影響しない。
