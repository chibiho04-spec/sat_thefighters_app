# 機材の重複（ダブルブッキング）警告 設計仕様

作成日：2026-06-20 / 対象：`案件管理アプリ/index.html`（単一HTMLファイル）

## 1. 目的
同じ機材が、撮影日が重なる複数の現場で所有数を超えて使われている場合に警告を出し、ダブルブッキングに気づけるようにする。

## 2. 決定事項（ユーザー確認済み 2026-06-20）
- **警告の場所**：ダッシュボード ＋ ワークシート編集画面の**両方**。
- **判定基準**：**数量も考慮（所有数超えで警告）**。
- **対象案件**：撮影日程が入っている全案件（削除済みは除外）。所有数は機材マスタ(`equip_master`)の `qty`（実データ上、単体=1、SD/電池等=2〜8 で在庫数として有効）。

## 3. 判定ロジック（共通関数 `getEquipConflicts()`）
- 各案件について：撮影日程(`getWsDisplayData(num).date` / `parseDateStr`)で使用日を展開、`loadWS(num).equip` の checked 機材を使用数（`sv.qty` 未設定はマスタ既定）で集計。
- **「日ごと × 機材ごと」に全案件の使用数を合計**し、`合計 > 所有数(master.qty)` かつ **2案件以上**が同日使用 → その機材＝重複（その日）。
  - 1案件単独で所有数超え（入力ミス等）は本警告の対象外（ダブルブッキングではないため）。
- 返り値：
  - `caseItems`: `{ 受注No: [重複機材id...] }`（各案件のどの機材が重複しているか）
  - `conflicts`: `[{ id, name, owned, maxUsed, days:[yyyymmdd...], cases:[受注No...] }]`（機材ごとの集約・バナー/警告文用）

## 4. 表示
### ダッシュボード
- 上部（ワークシート見出し付近）に赤バナー：`⚠️ 機材の重複 N件：SONY FX6／…`（表示中カードに関わる重複のみ。0件なら非表示）。
- 各カードの「使用機材」一覧で、`caseItems[num]` に含まれる機材名を**赤字＋⚠️**でハイライト。
  - `buildWsPreviewHTML(num, conflictIds)` に重複id集合を渡し、機材を id 付きで描画して該当を赤く。

### ワークシート編集画面
- 機材セクションの上に警告要素 `#wsd-equip-warning` を追加。`openWS(num)` で `getEquipConflicts()` を引き、`caseItems[num]` があれば赤い枠で：
  `⚠️ 他の現場と日程が重なる機材：SONY FX6（使用2/所有1・6/22）…` を表示（重複が無ければ空＝非表示）。

## 5. 影響範囲（壊さない）
- 追加が中心：`getEquipConflicts()` 新設、`buildWsPreviewHTML` に引数追加（後方互換・既定は警告なし）、`renderDashboard` でバナー＆id集合を渡す、`openWS` で警告描画、HTML/CSS追加。
- 既存の集計・保存・同期は変更しない。警告は**表示のみ**（データ書き換えなし）。
- グローバル名 `getEquipConflicts`/`renderWsEquipConflict` は新規・衝突なし。

## 6. テスト計画
- `node --check` 相当。
- 実機（同期OFF）：
  1. 同日に同じ単体機材を使う2案件 → 重複検出（バナー＆カード赤＆編集画面警告）。
  2. 所有数2の機材で合計2まで→警告なし、合計3→警告。
  3. 日程が重ならない場合は警告なし。1案件のみ所有数超えは対象外。
  4. ダッシュボード/編集画面の両方で表示・崩れなし。クリック等の既存動作に影響なし。

## 7. やらないこと（YAGNI）
- 自動の機材割り当て調整・予約管理。警告（気づき）のみ。
- 機材以外（クルー等）の重複検出。
