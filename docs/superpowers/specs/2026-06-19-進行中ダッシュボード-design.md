# 進行中ダッシュボード（PC専用・16:9モニター）設計仕様

作成日：2026-06-19 / 対象：`案件管理アプリ/index.html`（単一HTMLファイル）

## 1. 目的
会社のメインフロアのモニター（16:9）に「これから撮る現場」を一目でわかるよう常時表示する。カレンダーと、撮影前ワークシートのプレビューを並べて見せる。クリックで各案件の編集画面へ飛べる。スマホ非対応（PC/モニター専用）。

## 2. 方針（ユーザー確認済み 2026-06-19）
- **別アプリにせず、いまのアプリ内にPC専用ダッシュボードページを追加**。理由：データ・同期・`openWS`（編集遷移）等の既存資産を再利用でき最速、編集画面に直接飛べる、同期が自動で効く。
- **対象データ（現場案件）の定義**：`撮影日程あり ∧ 撮影日程の初日が今日より後（未来）∧ 未請求`。＝これから撮る現場。撮影日が近い順。
  - カレンダー・ワークシートタイルとも**この同じ集合**を見せる。
- **撮影前ワークシートが多いとき**：小さめタイルをグリッドで並べる（ローテーションなし）。入りきらない分は直近優先で表示し「ほか◯件」を出す。
- **操作**：表示中心。カード/日付をクリックすると `openWS(受注No)` で編集画面へ。
- **16:9・スマホ非対応**：狭い画面では「PC／モニターでご覧ください」の案内のみ。

## 3. 画面構成（16:9・全画面オーバーレイ）
ルート要素 `#dashboard-screen`（`position:fixed; inset:0; z-index:11000;` 程度の最前面オーバーレイ。phone-frame の外、ビューポート全体を占有）。背景はダークネイビー。レイアウトは縦3段：

1. **上部バー**（高さ固定）：左「✊ サット 進行中ボード」／右に `進行中 N件`（=対象集合の件数）・`今週の現場 M件`（初日が今日〜7日以内）・日付・時計（毎秒更新）・最終同期時刻。
2. **本体**（残り全部・横2分割）：
   - **左カラム（約38%）＝月間カレンダー**：当月を表示。対象集合の案件を撮影日（各日）に色付きチップで配置。今日を強調枠。チップ/日付クリックで `openWS`。月送りは不要（常に当月固定。日付変化で自動的に当月へ）。
   - **右カラム（約62%）＝これからの現場ワークシート（撮影日が近い順）**：対象集合を**小さめタイルのグリッド**（`repeat(auto-fill, minmax(...))` で4〜6枚目安）。各タイル＝白い「紙」風で、受注No／進捗バッジ／📅撮影日程／件名／取引先／**クルー**／**機材**。クリックで `openWS`。入りきらない分は末尾に「ほか◯件」。
3. **下部バー**（細い）：「🖥 16:9 モニター専用」「5分ごとに自動更新」等の補助表示。

## 4. データ取得（読み取り専用・既存関数を再利用）
- 対象集合 `getDashboardCases()`：
  - `getSavedProjects()` を走査し、`getDeletedWS()` を除外。
  - 各案件の撮影日程（`getWsDisplayData(num).date` または ws_ の `wsd-shootdate` / project.date）を `parseDateStr()` で配列化し、**初日**を取得。
  - 条件：初日が今日より後（未来）。撮影日程が空/解析不可は除外。
  - 未請求：`loadInvStatus(num).billed` が false（請求済/入金済は除外）。
  - 初日昇順でソート。
- タイル表示項目：
  - 受注No、進捗バッジ（`calcWsProgress(num)` → 受付/準備中/撮影中/請求書作成中。未来の現場は受付/準備中想定）。
  - 撮影日程テキスト（`getWsDateText(num)`）。件名・取引先（`getWsDisplayData`）。
  - クルー（`getWsCrewList(num)` → "C:川崎" 等の配列）。
  - 機材：**新規ヘルパー `getWsEquipNames(num)`** を追加。ws_ の `saved.equip`（id→数量/真偽）と `equip_master`（id→name）から機材名リストを作る（数量>0 のものを名前で列挙、先頭数件＋「ほか」）。
- カレンダー：対象集合を撮影日の各日に配置。日付→案件の対応は `parseDateStr` で各案件の全日を展開して当月分をマッピング。

## 5. 起動・アクセス・自動ログイン
- **人が開く**：管理室（または会計室／ホーム）に「🖥 ダッシュボード」ボタンを設置 → `openDashboard()`。ログイン済みユーザーはそのまま、クリックで編集も可。
- **モニター用URL**：`…/index.html#dashboard`。ロード時に `location.hash === '#dashboard'` を判定し、**PIN入力なしの表示モードで自動起動**（社内モニター前提）。未ログインでも `openDashboard()` を呼び、ログイン画面はスキップ。クリックで `openWS` も開ける。
  - 自動ログインの実装は、ログインゲートを通さずダッシュボードを最前面表示する形（既存 `loginSuccess` は呼ばず、表示専用に必要な初期化＝同期取り込み・描画のみ実行）。`#dashboard` 以外は従来どおりログイン必須。
- **閉じる**：`closeDashboard()`（ボタン経由で開いた場合に戻る。モニター運用では閉じない想定）。

## 6. 自動更新
- 時計：1秒ごとに時刻表示のみ更新（`setInterval`）。
- データ：既存の同期（起動時／5分ごと／ネット復帰）に乗せ、**同期完了フックで `renderDashboard()` を再実行**（開いているときのみ）。加えて保険として数分ごとに再描画。
- 日付跨ぎ：再描画時に「今日」を取り直すので、深夜0時を跨いでも当月/今日強調が自然に更新される（次の再描画タイミングで反映）。

## 7. レスポンシブ / スタイル
- `#dashboard-screen` はビューポート全体（`100vw×100vh`）。16:9 を基準にしつつ、実モニター比率に合わせて中身は flex/grid で伸縮（レターボックスはしない）。
- 文字はモニター遠目想定で大きめ。タイル内の最小文字は読める範囲で。
- 幅が狭い（例 `max-width: 900px`）ときは中身を隠し「PC／モニターでご覧ください」の案内のみ表示（スマホ非対応）。
- font_scale（`_captureAndApplyFs`）の対象外にする（`#dashboard-screen` を除外リストに追加）。ダッシュボードは独自サイズで管理し、ユーザーの文字サイズ設定に引っ張られない。

## 8. 影響範囲（他機能を壊さない）
- 追加が中心：新オーバーレイ `#dashboard-screen`、新関数（`openDashboard`/`closeDashboard`/`renderDashboard`/`getDashboardCases`/`getWsEquipNames`/`renderDashCalendar`/`renderDashWorksheets`）、起動時 `#dashboard` 判定、ボタン1つ、同期完了フックへの1行追加。
- **既存の `getSavedProjects`/`getWsDisplayData`/`getWsCrewList`/`calcWsProgress`/`loadInvStatus`/`parseDateStr`/`openWS` は変更しない**（流用のみ）。
- 書き込み・同期pushはダッシュボードからは行わない（読み取り専用）。編集は `openWS` 先で従来どおり。
- グローバル関数名の衝突なし（dash 接頭辞で命名）。
- z-index：最前面（編集画面 175・モーダル 10000 より上の 11000 台）。ダッシュボードを開いた状態で `openWS` すると編集画面(175)はダッシュボードの下になるため、`openWS` 時はダッシュボードを一時的に隠す（または編集画面を最前面に。実装時に決定：ダッシュボードを `display:none` にして編集、戻るでダッシュボード復帰が単純）。

## 9. テスト計画（実装後）
- `node --check` 相当（script 抽出構文チェック）。
- 実機（同期OFF `window._gasUrlOne=()=>''`）：
  1. 対象集合の正しさ：未来の撮影日 ∧ 未請求 のみ。過去・請求済・撮影日なしが出ない。初日昇順。
  2. タイル表示：受注No/進捗/日程/件名/取引先/クルー/機材が出る。多いとき小タイル＋「ほか◯件」。
  3. カレンダー：当月に対象案件が撮影日へ配置、今日強調。
  4. クリック→`openWS` で編集画面が前面に開く。戻るとダッシュボード復帰。
  5. 時計が動く。同期完了で再描画される（同期完了フックを手動発火して確認）。
  6. `#dashboard` URL で自動起動（ログインスキップ）。`#dashboard` 無しは従来どおりログイン必須。
  7. 16:9（例 1920×1080 / 1280×720）で崩れない。狭い幅で「PCでご覧ください」。
- デバッグ：対象集合の定義・既存機能無影響・自動起動の安全性を多観点レビュー＋裏取り。

## 10. やらないこと（YAGNI）
- スマホ／タブレット最適化レイアウト（PC専用）。
- ダッシュボードからの直接編集（編集は `openWS` 先で行う）。
- 月送り・期間切替（常に当月＋未来の現場）。
- 撮影中/過去の現場・請求済案件の表示（対象外）。
- ローテーション表示（小タイルグリッドで対応）。
