# 請求書テンプレート機能 設計仕様

作成日：2026-06-30 / 対象：`案件管理アプリ/index.html`（単一HTMLファイル）

## 1. 目的
定期的にくる案件（例：報道技術協力AorCA／琉球朝日放送 の月次請求）で、毎回ゼロから請求書を作る手間をなくす。
請求書の「ひな形（テンプレート）」を保存し、そこから新しい請求書をワンタッチで起こせるようにする。

## 2. 決定事項（ユーザー確認済み 2026-06-30）
- **保存先**：この端末のブラウザ内のみ（同期しない）。後で全端末共有を追加できる作りにする。
- **テンプレートに残す情報（文字）**：取引先・宛先（御中/様）・郵便番号・住所・件名・自社担当・備考・書類モード（請求書/見積書）・明細の各行の「品目名／単位／課税区分／外注フラグ／商品コード」。
- **テンプレートから作成時に空にする（毎回入力する）もの**：受注番号（自動採番）・請求日（＝今日）・作業日（空）・明細の **数量** と **単価（金額）**・値引き（0）・外注費の金額。外注費は「名目（外注先名）」のみ残し金額は空。
- **含めない**：請求済/入金済フラグ（テンプレートは常に未請求の新規として開く）。
- **入口（UI）**：
  - 請求書タブ上部に「📑 テンプレート」ボタン → テンプレート一覧画面。
  - 請求書編集画面に「📑 テンプレートとして保存」ボタン（「納品書化」ボタン付近）。

## 3. データ
### localStorage キー：`inv_templates`（JSON配列）
```javascript
[{
  id: 'tpl_1750000000000',          // 一意ID。ブラウザ実行なので 'tpl_' + Date.now() で生成（保存ごとに固定）
  name: '報道技術協力AorCA（琉球朝日放送）月次', // 表示名
  company: '琉球朝日放送株式会社',
  contact: '報道制作局長兼報道制作部長　棚原勝也',
  honorific: '御中',                 // 御中 / 様
  zip: '', addr: '',
  title: '報道技術協力AorCA',
  staff: '川崎哲也',
  memo: '',
  docMode: 'invoice',               // invoice / estimate
  lines: [                          // 数量・単価は空（毎回入力）
    { code:'', name:'カメラマン人件費', unit:'日', qty:'', price:'', taxable:true, outsource:false }
  ],
  outsourceList: [                  // 外注費：名目のみ・金額は空
    { name:'', cost:'', taxable:false }
  ],
  _updatedAt: '2026-06-30T00:00:00.000Z'
}]
```
- 売上集計には一切関与しない（`inv_` キーとは別キー。`_collectSalesItems` の対象 `^(INV-\d|[TEP]\d)` にも当たらない）。
- 同期対象外（`SYNC_ENABLED` に追加しない）。

## 4. 関数（すべて index.html に追加。既存名と衝突しない接頭辞）
- `loadInvTemplates()` / `saveInvTemplates(list)` — 配列の読み書き。
- `_collectInvEditorContent()` — **現在の請求書編集画面の内容**（宛先・件名・担当・備考・明細・外注費・値引き・docMode）を1オブジェクトに収集する共通関数。
  - 既存 `createDeliveryFromCurrent` と `saveInvDoc` が同じ収集処理を重複して持っているので、この共通関数に切り出して両者からも呼ぶ（DRY。挙動は変えない）。
- `saveCurrentAsTemplate()` — `_collectInvEditorContent()` で集めた内容から、数量・単価・外注金額・値引きを空にした「ひな形」を作り、`prompt` で名前を聞いて `inv_templates` に追加保存。完了をスピーチバブルで通知。
- `openInvTemplateBox()` / `closeInvTemplateBox()` — テンプレート一覧オーバーレイ `#inv-template-screen` の開閉。
- `renderInvTemplateList()` — 一覧描画。各行：テンプレ名＋「この内容で作成」「✏️ 名前変更」「🗑 削除」。0件なら空状態の案内。名前は `escapeHtml()` を通す。
- `createInvoiceFromTemplate(id)` — テンプレートの内容を使って **新規請求書編集画面を事前入力で開く**。
  - `_openBlankInvScreen('create')` と同じ初期化（`_invIsNew=true`、`_invRemoteChecked=false`、受注番号＝`getNextOrderNum('T')` の新番号、請求日＝今日）を行い、その上にテンプレの宛先・件名・担当・備考・docMode・明細（数量/単価は空）・外注費（金額は空）を流し込む。
  - 受注番号は編集可（新規時）。保存時は既存の `saveInvDoc` がそのまま走る＝**先日入れた他端末との重複チェック（`_verifyOrderNumFreeRemote`）も自動で効く**。
  - テンプレート一覧は閉じてから編集画面を前面に出す。
- `renameInvTemplate(id)` — `prompt` で改名。
- `deleteInvTemplate(id)` — `showConfirm` で確認してから削除。

## 5. UI / レイアウト
- **請求書タブ上部**：既存の「＋新規請求書」やフィルター付近に「📑 テンプレート」ボタンを1つ追加（`onclick="openInvTemplateBox()"`）。
- **テンプレート一覧オーバーレイ `#inv-template-screen`**：
  - 全画面オーバーレイ。ヘッダーに「📑 請求書テンプレート」と「◀ 戻る」。
  - **既知ルール厳守（2026-06-09）**：新しい全画面オーバーレイは `@media (min-width:769px)` と `(min-width:1200px)` の `#xxx-screen` ルール一覧に `top:54px/left:200px` を追加すること。
  - z-index は既存の編集画面（175）より上に出ない値で良い（一覧→作成で編集画面を前面にするため、一覧は編集画面より下でよい。実装時に既存パターンに合わせる）。
- **請求書編集画面**：「納品書化」ボタンの並びに「📑 テンプレートとして保存」ボタンを追加。
- 文字サイズ（`font_scale`）追従は既存の編集画面と同じ扱い。固定px幅があれば `--fs-scale` で追従。

## 6. データフロー
1. 過去/編集中の請求書を開く → 「📑 テンプレートとして保存」→ 名前入力 → `inv_templates` に1件追加。
2. 請求書タブ →「📑 テンプレート」→ 一覧 →「この内容で作成」。
3. 新規請求書が事前入力で開く（番号自動・請求日今日・数量/金額空）。
4. 数量・金額・作業日などを入力 → 通常保存（採番の重複チェック込み）→ 売上にも計上。

## 7. 影響範囲（壊さない）
- 追加が中心。`saveInvDoc` / `createDeliveryFromCurrent` は明細収集を共通関数 `_collectInvEditorContent()` に切り出すリファクタのみ（出力は同一になるようにし、既存テストで挙動不変を確認）。
- 売上集計・同期・採番ロジックは不変。テンプレは別キーで売上・同期に出ない。
- 新規グローバル関数名（`loadInvTemplates`/`saveInvTemplates`/`saveCurrentAsTemplate`/`openInvTemplateBox`/`closeInvTemplateBox`/`renderInvTemplateList`/`createInvoiceFromTemplate`/`renameInvTemplate`/`deleteInvTemplate`/`_collectInvEditorContent`）は既存と重複なしを確認のうえ使用。

## 8. テスト計画（CLAUDE.md 必須工程に準拠）
- `node --check` 相当の構文チェック（script 抽出）。
- 実機（同期OFF含む）：
  1. 既存の請求書を開いて「テンプレートとして保存」→ `inv_templates` に保存される。名前が出る。
  2. 「📑 テンプレート」一覧に表示。0件時の空状態も確認。
  3. 「この内容で作成」→ 新規請求書が事前入力で開く。**受注番号は自動採番・請求日は今日・数量と金額は空・宛先/件名/品目名は入っている**。
  4. そのまま数量・金額・作業日を入れて保存 → 通常の請求書として保存され、売上表にも出る。**テンプレ自体は売上に出ない**。
  5. 受注番号の重複チェック（`_verifyOrderNumFreeRemote`）がテンプレ経由作成でも効く。
  6. 名前変更・削除（確認ダイアログ）。
  7. リファクタ後 `createDeliveryFromCurrent`（納品書化）と通常保存が従来どおり動く（明細・外注費・値引きが正しく収集される）。
  8. **UI 4観点**：スマホ幅(390px)／デスクトップ × 文字100%／150%。一覧・ボタン・編集画面の崩れ/はみ出しなし。オーバーレイがPCで `top:54px/left:200px` に出る。
- 完了報告に「✅ 構文OK」「✅ 確認した関連箇所」を明記。Git push まで実施。

## 9. やらないこと（YAGNI）
- 端末間同期（後日、`inv_templates` 用シート＋kind追加＋GAS再デプロイで対応可能）。
- テンプレートのフォルダ分け・カテゴリ・並び替え。
- 毎月の自動生成・スケジュール。
- 案件（ワークシート）からのテンプレ化（今回は請求書編集画面からのみ）。
- 請求日・受注番号の自動連番予約（採番は既存ロジックに任せる）。
