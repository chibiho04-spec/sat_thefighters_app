# 請求書ドキュメント改良（空行・行備考・3書類の見分け）設計仕様

作成日：2026-07-03 / 対象：`案件管理アプリ/index.html`（単一HTMLファイル）

## 1. 目的
請求書まわりの3つの改良を、請求書ドキュメント（編集画面＋印刷/プレビュー）に加える。
1. **空行**：明細の項目間にスペース（空行）を入れて見やすくする。
2. **行備考**：明細の各行に備考を書けるようにし、プレビューの右端「備考」列に表示。
3. **3書類の見分け**：請求書／見積書／納品書を、左上の色帯＋アイコンと右上の枠つきバッジで区別。白黒印刷でも判別できる。

## 2. 決定事項（ユーザー確認済み 2026-07-03）
- **① 空行の入れ方**：各明細カードの「＋ 下に空行」ボタンで、その行の直後に空行を挿入（並べ替え不要）。
- **② 行備考**：明細編集モーダルに「備考」欄を追加。プレビュー右端「備考」列に表示（今その列にある「非課税」表示と**共存**）。
- **③ 見分け**：左上に色帯（緑=請求 `#1f9e6f`／橙=見積 `#d9791d`／青=納品 `#2377ba`）＋アイコン、右上に黒枠バッジ（書類名）。白黒でもアイコン＋バッジで区別。本文レイアウトは3書類共通のまま。

## 3. データモデル変更
明細1行に2フィールドを追加。1行 = `{ code, name, unit, qty, price, taxable, outsource, note, spacer }`。
- `note`（string）：行の備考。
- `spacer`（boolean）：空行フラグ。`true` の行は摘要・数量・単価なしの空行で、**集計・課税・外注に一切関与しない**。
- **後方互換**：既存データに `note`/`spacer` が無くても壊れない（未定義は `note=''` / `spacer=false` として扱う）。

## 4. アーキテクチャと変更箇所（すべて index.html）
明細は二層構造：
- **隠しテーブル** `#inv-lines-body` の `.inv-line-row`（＝**データの保管源**。`_collectInvLines`/`buildDocHTML` が読む）
- **カードUI** `#inv-cards`（`renderInvLineCards`）＋**編集モーダル** `#inv-line-modal`（＝**見た目・編集**）

保管は隠しテーブルの各 `tr` の **data 属性**で行う：`tr.dataset.note`（備考）／`tr.dataset.spacer`（'1' なら空行）。

### 4-1. `renderInvLines(masterLines, savedLines, readonly, discount)`（隠しテーブル生成）
- 各 `tr` に `dataset.note = line.note || ''` を設定。
- `line.spacer` が true の行は `tr.dataset.spacer='1'` を付け、`inv-line-row` は付けるが**編集inputは出さず**、摘要/数量/単価は空（値の保管のみ）。`inv-amount-*` は空。
- 通常行は従来どおり input を生成し、`dataset.note` を保持。

### 4-2. `_readInvLineRows()` / `_collectInvLines()`（行の読み取り）
- 各行で `note = tr.dataset.note || ''`、`spacer = tr.dataset.spacer === '1'` を読み、返すオブジェクトに含める。
- `_collectInvLines()` の除外条件 `if (!name && !price) return;` を **`if (!spacer && !name && !price) return;`** に変更（**空行は残す**、通常の空行は従来どおり捨てる）。返り値に `note`・`spacer` を含める。
- `_readInvLineRows()` も同様に `note`・`spacer` を含める。

### 4-3. `renderInvLineCards()`（カードUI）
- **通常カード**：`note` があれば小さなグレー文字で1行表示（`_esc(note)`）。各カードのフッターに「**＋ 下に空行**」ボタンを追加 → `insertSpacerAfter(i)`。
- **空行カード**：`spacer` の行は `— 空行 —` の細いカード（削除ボタンのみ、タップ編集なし）。
- 読み取り専用（`_invReadonly`）時は「下に空行」「削除」を出さない。

### 4-4. 編集モーダル `#inv-line-modal`
- 「摘要」の下に「**備考**」欄（`<textarea id="inv-modal-note">`）を追加。
- `openInvLineModal(idx)`：`tr.dataset.note` を復元。`saveInvLineModal()`：入力値を対象行の `dataset.note` に保存。
- 空行はモーダルを使わず、`insertSpacerAfter` / カード削除で操作。

### 4-5. 新規関数 `insertSpacerAfter(i)`
- 現在の行配列を `_readInvLineRows()` で取得 → `i` の直後に `{ code:'', name:'', unit:'', qty:0, price:0, taxable:true, outsource:false, note:'', spacer:true }` を splice 挿入 → `renderInvLines([], rows, false, 現在の値引き)` で再描画。
- 現在の値引きは `document.getElementById('inv-discount').value` から読む。

### 4-6. 印刷HTML `buildDocHTML(type, emptyRows)`
- **行備考**：6列目「備考」を `note` ＋ 非課税マーカーの併記に変更。
  - `note` があれば通常文字で表示。`taxable === false` なら赤字小文字「非課税」を（noteの下に）追記。両方空なら空セル。
  - 「備考」列幅を 60px → 90px に広げて折り返しやすく。
- **空行（spacer）**：明細ループの `if (!name) return;` を、**spacer 行は空欄の行として出力**するよう変更（6セル `&nbsp;`、`rowCount` は増やさない＝縞や最低行数計算には数えない）。集計には元から不参加。
- **③ 見分け**：`_T[type]` に `barColor`・`badge`（＝書類名）・`iconSvg`（**印刷用の小さなインラインSVG**。Tabler webフォントは印刷ウィンドウに無いため）を追加。ドキュメント最上部（現在の「1行目：空白」の位置）に差し込む：
  - **左上**：`barColor` 背景・白文字の帯（高さ約7mm）に `iconSvg`（白）＋書類名。
  - **右上**：黒枠（2px）＋黒文字のバッジ（書類名）。
  - 白黒対応：バッジは黒枠＋黒文字、アイコンも黒で出るため色に依存しない。帯の色が飛んでもバッジ＋アイコンで判別可。既存の `print-color-adjust:exact` で色は保持。
- 本文（宛先・表・合計・振込先）のレイアウトは3書類共通のまま。

### 4-7. 保存・テンプレ・同期
- `saveInvDoc`：明細収集は `_collectInvLines()` 経由なので `note`・`spacer` は自動で `status.lines` に入る（追加変更不要）。
- テンプレ：`saveCurrentAsTemplate`（`note` は残す・`spacer` も残す。数量/単価を空にする既存処理は通常行のみ）、`createInvoiceFromTemplate`（`note`・`spacer` を引き継ぐ。数量/単価は空表示のまま）。
- 同期（invoice kind）は `status` を丸ごとJSONで送るため、追加対応不要。

## 5. やらないこと（YAGNI）
- 明細の自由なドラッグ並べ替え（空行は insert-after で足りる）。
- 行ごとの色分け・書式（備考は文字のみ）。
- 3書類で表本文のレイアウト自体を変える（ヘッダーの帯／バッジ／アイコンで区別）。

## 6. テスト計画（CLAUDE.md 必須工程に準拠）
- `node` 構文チェック（script 抽出）。
- 実機（ローカル `http://localhost:8793/`）：
  1. モーダルで備考を入力→保存→**プレビュー備考列に表示**（非課税とも共存）。
  2. カードの「＋下に空行」→ 項目間に空行が入る→**プレビューで空欄行**→保存後も残る→**合計金額は不変**。
  3. 請求書/見積書/納品書で**帯色・アイコン・右上バッジ**が変わる。色を殺した（白黒）状態でも**バッジ＋アイコンで判別**できる。
  4. テンプレ保存→作成で**備考・空行が引き継がれる**（数量/単価は空）。
  5. **既存データ（note/spacer 無し）が壊れない**（従来の請求書を開いて崩れない）。
  6. **UI 4観点**（スマホ390px／デスクトップ × 文字100%／150%）：モーダルの備考欄・空行カード・「下に空行」ボタンが崩れない。
