# 受注ノート一覧ページ 設計仕様

作成日：2026-06-18 / 対象：`案件管理アプリ/index.html`（単一HTMLファイル）

## 1. 目的
新規案件作成で登録した案件を、受注番号を先頭に1案件＝1行で一覧できる「受注ノート」ページを会計室に追加する。受注番号の頭文字(T/E/P)などで並び替えできる台帳ビュー。**読み取り専用**（既存データを表示するだけで書き換えない）。

## 2. 決定事項（ユーザー確認済み 2026-06-18）
- **場所**：会計室タブ（tab-1）の「書類作成」セクション、**ワークシートボタンの上**に「📒 受注ノート」ボタンを新設。押すと全画面の一覧ページ（スライドイン）を開く。
- **1行に表示**：受注番号（先頭・太字・T/E/Pで色分け）｜ 案件名 ｜ 取引先 ｜ 撮影日程 ｜ 分類 ｜ 種別 ｜ 自社担当。できるだけ1行、狭い画面では自然に折り返し。
- **行クリック**：その案件のワークシート詳細を開く（`openWS(num)`）。
- **並び替え**：(1)受注番号の頭文字 T/E/P順（T→E→P、それ以外は末尾） (2)受注番号 新→古 (3)受注番号 古→新 (4)受付日 新→古 (5)撮影日 新→古 (6)取引先 名前順。
- **検索**：受注番号・件名・取引先・自社担当で絞り込み。**件数**表示。

## 3. データ源（読み取り専用）
- `getSavedProjects()`（projects_saved）の各案件。`getDeletedWS()` に含まれる番号は除外。
- 表示値は既存の `getWsDisplayData(num)`（title/company/date/type/orderType を保存済み編集値優先で解決）を流用。自社担当は project.staff、受付日は project.receivedAt を使用。
- **書き込みは一切行わない**（同期push・localStorage変更なし）。→ 他機能への影響リスクが構造的に小さい。

## 4. UI 構成
- 新規オーバーレイ `#order-note-screen`（`worksheet-list-screen` と同じスライドイン方式：position:absolute; inset:0; transform で開閉）。
  - ヘッダー：◀戻る（closeOrderNoteScreen）／タイトル「📒 受注ノート」／件数。
  - ツールバー：検索入力（`order-note-search`）＋並び替え `<select id="order-note-sort">`。
  - 本文：`#order-note-list`（wsd-scroll でスクロール）。月別等の区切りは無し（並び替え主体）。
- **デスクトップ配置**：CLAUDE.md のルールに従い、`@media (min-width:769px)` と `(min-width:1200px)` の `#xxx-screen` 一覧に `#order-note-screen` を追加（top:54px/left:200px）。

## 5. 1行レイアウト
- flex 行。受注番号（固定幅・太字・T=青/E=緑/P=紫などで色分け）＋ 件名（flex:1・省略表示）＋ 取引先（省略）＋ 撮影日程 ＋ 分類 ＋ 種別 ＋ 自社担当（小さめテキスト/バッジ）。
- `escapeHtml()` で全テキストをエスケープ。onclick の num はエスケープ＋シングルクォート安全に。
- 狭い画面（スマホ）では主行（番号＋件名）＋副行（取引先・日程・分類・種別・担当）に自然に折り返す。

## 6. 並び替えロジック
- TEP順：頭文字を T=0,E=1,P=2,その他=9 に写像して昇順、同순位は受注番号で昇順。
- 受注番号順：文字列比較（localeCompare, 'ja'）昇順/降順。
- 受付日：project.receivedAt（YYYY-MM-DD）で比較。無しは末尾。
- 撮影日：`parseDateStr(date)` の最初の日付で比較（範囲対応）。無しは末尾。
- 取引先：company を localeCompare('ja') 昇順。
- 既定は「受注番号の頭文字 T/E/P順」。

## 7. 関数（新規・既存は変更しない方針）
- `openOrderNoteScreen()` / `closeOrderNoteScreen()`：表示/非表示（transform 切替）。
- `renderOrderNoteList()`：収集→検索フィルタ→並び替え→行HTML生成→件数更新。
- `onCrewSel` 等の既存は触らない。`_guardedClose`/ESC やスクリーンクローズ一覧（`{ id: ... }` の配列, 8243付近）に `order-note-screen` を追加（戻る/閉じる挙動の一貫性のため）。

## 8. 回帰・影響チェック（ユーザー要望：他機能に影響がないか確認）
- **新規追加が中心**：新ボタン1つ＋新オーバーレイ＋新関数。既存関数は変更しない（流用のみ：getSavedProjects/getDeletedWS/getWsDisplayData/parseDateStr/openWS/escapeHtml）。
- **会計室タブ(tab-1)のHTML**にボタンを1つ足す→他ボタン（請求書/見積/納品/売上表/受注番号変更/重複整理/ワークシート）の配置・動作が崩れないか確認。
- **z-index/オーバーレイ**：`#order-note-screen` の重なりが既存画面（ワークシート一覧/詳細/売上/モーダル群）と衝突しないか。開いた状態で行クリック→openWS（詳細 z-index 175）が前面に出るか。
- **@media 追加**：デスクトップ/スマホ×文字100%/150% の4パターンで一覧・他画面が崩れないか。
- **font_scale**：px指定が二重拡大/はみ出ししないか（既存ルール準拠）。
- **同期・データ**：本ページは読み取り専用。push/localStorage書き込みが無いことをコード確認。
- **グローバル名の衝突**：新関数名（openOrderNoteScreen等）が既存と被らないこと。

## 9. テスト計画（実装後）
- `node --check` 相当の構文チェック。
- 実機（同期OFF `window._gasUrlOne=()=>''`）：
  1. 会計室にボタン表示・ワークシートの上に配置。
  2. ページが開く／件数・行が出る／受注番号先頭・1行レイアウト。
  3. 並び替え各種（特に T/E/P順）が効く。
  4. 検索フィルタ。
  5. 行クリック→ワークシート詳細が前面に開く。
  6. スマホ幅/PC×文字100%/150% で崩れ無し。
  7. **回帰**：会計室の他ボタン全部が従来どおり動く／ワークシート一覧・売上表・各モーダルが正常。
- デバッグチーム（データ正確性／表示崩れ/回帰の並列レビュー＋裏取り）。

## 10. やらないこと（YAGNI）
- 受注ノートからの編集・新規作成・削除・番号変更（閲覧専用。編集は行クリック先のワークシートで行う）。
- CSV出力・月別区切り（今回は並び替え主体。必要なら別途）。
- 標準INV-/単独請求書の表示（対象は新規案件作成由来の案件のみ）。
