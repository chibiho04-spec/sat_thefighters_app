# 同期を1個にまとめる — セットアップ＆移行手順（初心者向け）

作成日: 2026-06-01
対象: Sat the Fighters の同期（案件・子・取引先・商品・メディア・機材・請求書・スタッフ）を、1個のスプレッドシートにまとめる

## これは何？

今までは「案件」「機材」「取引先」…と**8個ものスプレッドシート＋8個のURL**を別々に設定していました。
これを **1個のスプレッドシート ＋ 1個の Apps Script ＋ URL 1個 ＋ 合言葉1個** にまとめます。

- データは**各端末のローカルに全部残っています**。新しいシートへ「全部送信」すれば移せます。**消えません。**
- 1個のスプレッドシートの中に、データの種類ごとに**タブ（シート）**が自動で作られます（案件・子・取引先・商品・メディア・機材・請求書・スタッフ）。
- URL を設定しなければ、今までどおり**その端末のローカルだけ**で普通に使えます（同期オフ）。

---

## 手順の全体像（順番が大事）

1. 新しい**空のスプレッドシートを1個**作る。
2. 拡張機能 → **Apps Script** に `unified-sheet-Code.gs` の中身を貼り付け。
3. `SYNC_SECRET` を、**今までと同じ合言葉**に書き換える。
4. **ウェブアプリとしてデプロイ**して URL を1個コピー。
5. アプリの「データ管理」で、**同期URL** にその URL を貼り付けて保存。
6. データを持っている**1台だけ**で「🚀 新シートへ全部送信（初回シード）」を押す。
7. 他の端末は同じ URL を入れて起動 → 約1.5秒後に**自動で取り込み**。

---

## 1. スプレッドシートを作る

1. Google ドライブで新しいスプレッドシートを作成します（名前は「Sat統合同期」など分かりやすいもの）。
2. **タブ（シート）は何も用意しなくて大丈夫です。** 初回アクセス時にスクリプトが
   「案件」「子」「取引先」「商品」「メディア」「機材」「請求書」「スタッフ」の各タブと、
   その見出し行を**自動で作ります**。

> ⚠️ 旧8シートとは**別の新しいシート**です。今までのシートはそのまま残しておいてOK（保険）。

---

## 2〜4. Code.gs を貼る・合言葉・デプロイ

`staff-sheet-setup.md` と同じ要領です。

1. スプレッドシートのメニュー **拡張機能 → Apps Script** を開きます。
2. 最初からあるコードを全部消して、`docs/superpowers/unified-sheet-Code.gs` の中身を**まるごと貼り付け**ます。
3. 先頭近くにある
   ```js
   var SYNC_SECRET = 'ここに合言葉';
   ```
   の `ここに合言葉` を、**今まで他のシートで使っていたのと同じ合言葉**に書き換えます。
   （アプリ側の「合言葉(token)」と**完全に一致**させてください。）
4. 保存（フロッピーのアイコン）。
5. 右上の **デプロイ → 新しいデプロイ** をクリック。
   - 種類：**ウェブアプリ**
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
6. **デプロイ**を押し、表示された **ウェブアプリの URL を1個コピー**します
   （`https://script.google.com/macros/s/……/exec` の形）。

> ⚠️ コードを直したら、毎回 **デプロイ → デプロイを管理 → 編集（鉛筆）→ バージョン「新バージョン」→ デプロイ** で更新が必要です。

---

## 5. アプリに URL を登録

1. アプリ上部の「**データ管理**」を開きます。
2. 「📊 案件マスタ同期」の **同期URL** 欄に、さっきコピーした URL を貼り付けます。
3. 「💾 URLを保存」を押します。
4. すぐ下の**合言葉(token)** 欄に、Code.gs に書いたのと同じ合言葉が入っているか確認します（未入力なら入れて保存）。

> 旧バージョンの個別URL欄（親 / 子 / 取引先 / 商品 / …）は**廃止**しました。この URL 1つだけでOKです。

---

## 6. 初回シード（1台だけ）

データを一番たくさん持っている端末を**1台だけ**選び、そこで行います。

1. 「データ管理」を開きます。
2. 「🚀 新シートへ全部送信（初回シード）」を押します。
3. 確認ダイアログで **OK**。
4. しばらく待つと「✅ シード完了：◯件送信」と出ます。これで新シートに全データが入りました。

> ⚠️ 初回シードは**1台だけ**でOK。複数台で同時に押す必要はありません（後勝ちで上書きされます）。

---

## 7. 他の端末

1. 各端末で「データ管理」を開き、**同じ URL** と**同じ合言葉**を入れて保存します。
2. アプリを開く（または再読み込みする）と、**起動の約1.5秒後に自動で取り込み**ます。
3. 手動でやりたいときは「🔄 全部同期（取り込み＋送信）」を押します。

---

## 動作確認

1. A端末で案件を1件保存します。
2. B端末でアプリを開く（または「🔄 全部同期」）→ **同じ案件が出れば成功**です。
3. A端末で案件を**削除**→ B端末で同期 → **B端末からも消えれば**削除の伝播もOKです。
4. 機材・取引先・商品・メディア・請求書・スタッフでも同じように確認できます。

---

## しくみ・既知の限界

- 全タブ共通で「**更新日時で後勝ち（サーバ側 LWW）**」。受け取った行の更新日時がシートより新しいときだけ上書きします。
- 削除は「**削除フラグ**」で論理削除（行は残りますが、アプリには出ません）。
- 合言葉は URL のクエリに乗ります（社内向けツールとして許容しています）。
- 旧8シートは**保険として残してOK**。新シートで問題が無いことを数日確認してから、後で消しても構いません。
- 同期URLを設定しなければ、アプリは**ローカルのみ**で普通に動きます。
